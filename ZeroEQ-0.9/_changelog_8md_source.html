67  Use cppnetlib to implement HTTP server to support multiple clients
68 * [182](https://github.com/HBPVIS/ZeroEQ/pull/182):
69  * Clearer HTTP server registration methods:
70  * handle( serializable ) instead of add()
71  * handlePUT( serializable/event ) instead of subscribe()
72  * handleGET( serializable/event ) instead of register_()
73  * remove( serializable/event ) instead of unsubscribe() and unregister()
74  * HTTP server REST API
75  * /registry endpoint returns handled events/objects
76  * <registry-item>/schema endpoints return schema describing the
77  events/objects. For servus::Serializable objects the schema is provided
78  via getSchema(), for events the schema can be optionally provided via
79  handlePUT() and handleGET()
